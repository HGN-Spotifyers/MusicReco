Taking the modelling a step further than simply recommendating a block of new songs or selecting a single song from the PCA or NMF model, we've developed an innovative method of song smoothing. NMF and PCA does have an inherent order to their predictions - as they provide a closeness score as to how well a new prediction fits onto the seed playlist. This can be thought of as the song which most closely shares a similar playlist space. As mentioned before though there could be a large number of possible, similarly relevant songs and with a large playlist these could be musically diverse.

Randomly throwing musically diverse songs onto a playlist as suggestions could appear to be jarring to the user and lack a sense of journey and personalisation. Song smoothing is designed to counter-act this and order suggestions in such as way that the user experiences this journey of discovery. In this report we've designed a simple version of song smoothing which operates on the first two principal component space of song attributes.

Song smoothing takes the most recently played song from a playlist (in this case a random seed song for testing purposes) and then uses a route finding algorithm (akin to the traveling salespersons problem) to find an efficient path to traverse the 2D song attribute space. In finding an efficient path songs can then be recommended with a sense of structure and journey, travelling smoothly around the relevant suggestions rather than jarringly jumping around.

Here is the core code and results for song smoothing:

The code is built in an object oriented style where each track is represented as a node with spatial coordinates and a next_song attribute. The algorithm operates on a greedy principle in that it always finds the absolute closest point from the current node without any consideration of the global optimum.

# GREEDY TRAVELING SALESMAN SOLUTION
class song_node:
    '''
    Test song node 
    '''
    def __init__(self, name, coord):
        self.name = name
        self.x = coord[0]
        self.y = coord[1]
        self.next_song = None
        self.distance_to_next = None
        
        
def traverse_closest_node(node1, other_nodes):

    distances = []
    for node in other_nodes:
        cur_distance = calculate_node_to_node_distance(node1, node)
        distances.append(cur_distance)

    best_index = np.argmin(distances)

    closest_song = other_nodes[best_index]

    node1.next_song = closest_song
    node1.distance_to_next = distances[best_index]

    _ = other_nodes.pop(best_index)
    if len(other_nodes)>0:
        traverse_closest_node(closest_song, other_nodes)
    
def calculate_node_to_node_distance(node1, node2):
    x_distance = node1.x - node2.x
    y_distance = node1.y - node2.y
    distance = np.sqrt(x_distance**2 + y_distance**2)    
    return distance
    

class network_distance:
    
    def __init__(self):
        self.distance = 0
    
    def calculate_tree_distance(self, node):
        if node.distance_to_next:
            self.distance = self.distance + node.distance_to_next
        if node.next_song:
            self.calculate_tree_distance(node.next_song)
